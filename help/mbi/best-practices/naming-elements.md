# レポートと要素に名前を付ける

[!DNL Adobe Commerce Intelligence]で構築を開始する前に、Adobeでは成功の秘訣をいくつか共有したいと考えています。 指標やフィルターなどを作成する方法を把握することは重要ですが、必要な情報が見つからない場合や、曖昧な場合、あらゆる作業に意味があるとは限りません。

## 命名規則が重要な理由？ {#why}

計算列、指標、レポートに名前を付ける方法によって、様々なユーザーが[!DNL Commerce Intelligence] アカウントを簡単に操作できるようになります。 これらの機能に名前を付ける際には、次の3つのCに留意してください。

* **明瞭度** - レポートの内容、指標の内容などを一目で確認できます。
* **一貫性** – お客様（およびAdobe サポート チーム）がアカウント内の要素とレポートを簡単に見つけて理解できるようにします。
* **信頼性** – 他のデータドリブン型[!DNL Commerce Intelligence] ユーザーを刺激し、権限を与えるには、ユーザーがデータをどのように理解し、使用しているかに自信を持つ必要があります。

経験豊富で真の命名法のヒントをお読みください！

## 一般的なベストプラクティス {#general}

### 有意義である {#meaningful}

可能な限り具体的にしましょう。 例えば、国の場合、それが送料または請求国であるかどうかを知っていますか？ これはユーザーの市区町村ですか、それとも取引の市区町村ですか？

**不正な例：**
収益

これは漠然としていて、あまり教えてくれません。

**良い例：**
収益（基本総計+手数料）
ユーザーの輸送国

これらの例は特殊であり、混乱の可能性を減らします。

### 大文字と一致する {#capitalize}

[!DNL Adobe]は、適切な名詞スタイルの大文字を使用しない限り、大文字の先頭を小文字で使用することをお勧めします。 例えば、**ユーザーの注文番号**&#x200B;ではなく、**ユーザーの注文番号**&#x200B;です

これは本当に好みの問題ですが、覚えておくべきことは、選択するものとの一貫性を保つことです。

### エンティティの一貫性 {#entity}

多くの企業は、既に命名規則を策定しています。 設定した指標とディメンションは、他のデータベースやツールで使用される指標と一致させます。 例：

* ユーザー、顧客、メンバー、アカウントの違い
* 企業、アカウント、組織の違い
* 登録と制作の違い

### スペルと文法 {#spelling}

あなたのスペルを再確認し、それらの厄介な所有物を忘れないでください！

## グラフ {#charts}

[ グラフ ](../tutorials/using-visual-report-builder.md)を命名する場合、次の式に従うと最も便利です：**（データパースペクティブ） + （指標） + （期間） + （時間間隔）**

**不正な例：**
収益

これは我々に報告書について何も語らない、それは悪い事だ。

**良い例：**
過去30日間の累計収益（月別）

これは、レポートの内容が&#x200B;**正確に**&#x200B;であることを示します。これは素晴らしいことです。

## ダッシュボード {#dashboards}

ダッシュボードには、その中に含まれるレポートをテーマ別に表す名前を付ける必要があります。 例えば、ダッシュボードに収益と注文に関連する情報のみが含まれている場合は、**ストア名 – 収益と注文**&#x200B;のような名前を付けることを検討してください。

逆に、ダッシュボードが異なるレポートを試す場所である場合は、そのダッシュボードに&#x200B;**名前のサンドボックス**&#x200B;という名前を付けて、その中に含まれるレポートがドラフトであることを確認します。

## ディメンション（計算列） {#dimensions}

新しい[ ディメンション ](../data-analyst/data-warehouse-mgr/creating-calculated-columns.md)を命名する場合、次の式に従うと最も便利です：**（エンティティ） + （N） + （時間枠） + （計算） + （コメント）**。 例：

ユーザーの最初の30日間の収益
* ユーザーの注文番号
* ユーザーの注文番号（監査待ち）

## フィルターセット {#filterset}

[ フィルターセット ](../data-user/reports/ess-manage-data-filters.md)は、通常、含める情報または除外する情報を説明する方法で名前が付けられます。 例えば、フィルターセットに名前を付ける&#x200B;**注文品目**&#x200B;を使用すると、任意のユーザーがフィルターセットのロジックを表示し、ビジネス全体でカウントされる注文情報を決定する注文情報を把握できます。 フィルターセットは、計算列と指標の両方に適用でき、理解しやすくする必要があることを忘れないでください。

## 指標 {#metrics}

[指標](../data-user/reports/ess-manage-data-metrics.md)は、基本的に、定期的に回答を求める質問です。 過去1 ヶ月の注文数はいくつですか？ 顧客の平均生涯価値は何か？ 指標に名前を付けて、利用者が提供している回答を反映させることをお勧めします。 また、特定の店舗や部門向けに同じ指標をフィルタリングする場合は、そのようにラベル付けする必要があります。 例：

平均的な顧客LTV （最初の30日間）
ストア名 – 収益

最後に、同じ指標が、個々のユーザーの計算方法に応じて、異なるタイムスタンプで整理されることがあります。 その場合は、次の名前にタイムスタンプを必ず含めてください。

収益（出荷済み\_at）
収益（created\_at）

## まとめ {#wrapup}

スタイルと命名規則を早期に確立することで、[!DNL Commerce Intelligence] アカウントで成功を収めることができます。 明瞭性、一貫性、信頼性の3つのCを忘れないでください。
